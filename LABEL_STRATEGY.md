# Label Strategy

## Goal
Use labels to make triage, ownership, and release planning easy to understand at a glance.

## Suggested label groups
- `type:*` for bug, feature, docs, and maintenance work
- `area:*` for proxy, protocol, UI, platform, plugin, and docs areas
- `priority:*` for triage urgency
- `good first issue` and `help wanted` for contributor onboarding
- `security` for sensitive changes
- `release:*` for versioning and shipping work

## Principles
- Keep the set small and understandable.
- Make labels consistent across issues and pull requests.
- Avoid too many one-off labels that fragment triage.
